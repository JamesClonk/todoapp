User Manual


Description

todoapp is browser-based Todo.txt application.

A description of the Todo.txt file format can be found here: The Todo.txt Format


Commandline flags for todoapp

Flag Default Description
-f, --file, todo.txt Specify the Todo.txt file to use for reading and storing tasks from. If this parameter is not given, todoapp will use todo.txt from the current directory where it was started.
-c, --config, todoapp.config Specify the configuration file to use for todoapp. If this parameter is not give, todoapp will automatically create a new file todoapp.config in the current directory where it was started.
-p, --port, 4004 Specify the port where the todoapp webserver will listen to.

Configuration file for todoapp

{
  "TodoTxtFilename": "todo.txt",
  "DeleteWarning": true,
  "ClearWarning": true,
  "Colors": {
    "PriorityA": "#cc0000",
    "PriorityB": "#ee9900",
    "PriorityC": "#eeee00",
    "PriorityD": "#3366ff",
    "PriorityE": "#33cc33",
    "PriorityF": "#cccccc"
  }
}

Security

todoapp intentionally does not provide any security mechanism to restrict access to it. It (tries to) follows the Unix philosophy of being short, simple, clear, modular, and composable.

If you wish to restrict access to todoapp only for localhost, you can simply setup the following iptables filter rule:

iptables -A INPUT -i lo -p tcp --dport 4004 -j ACCEPT
iptables -A INPUT -p tcp --dport 4004 -j DROP

Similarly, if you want to run todoapp on a remote host and have it reachable by the outside world, you could for example setup nginx as a reverse proxy in front of todoapp and configure it to use HTTP Basic Auth.
