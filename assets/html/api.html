API Guide

todoapp provides the following resources and HTTP methods on them:

URI Method Description
/api/task/:id GET Returns the task identified by given task id.
/api/task POST Creates a new task with sent form / json data and adds it to the list. Afterwards it will store the updated list into the defined todo.txt file.
/api/task/:id PUT Updates the task identified by given task id with sent form / json data. Afterwards it will store the updated list into the defined todo.txt file.
/api/task/:id DELETE Deletes the task identified by given task id from the list. Afterwards it will store the updated list into the defined todo.txt file.
/api/tasks GET Returns a complete list of all tasks. The data is read from a todo.txt file, as defined per the configuration setting TodoTxtFilename, or by the commandline switch --file.
/api/tasks DELETE Removes (only) closed / completed tasks from the list. Afterwards it will store the updated list into the defined todo.txt file.
/api/config GET Returns the current configuration settings. By default these are read and stored in todoapp.config in the directory where the application was started. This filename can also be specified with a commandline switch, --config.
/api/config PUT Updates the configuration settings. This will store the settings into the todoapp.config file.
